# 书驿云桥项目部署文档

## 1. 项目概述

书驿云桥是一个高校图书资源共享生态平台，旨在促进高校间图书资源的共享与流通，减少资源浪费，提高图书利用率。

## 2. 环境准备

### 2.1 系统要求

- **前端**：Node.js 18+，npm 9+
- **后端**：Python 3.10+，pip 23+
- **数据库**：MySQL 8.0+
- **缓存**：Redis 7.0+
- **Web服务器**：Nginx 1.18+
- **容器化**：Docker 20.10+，Docker Compose 1.29+

### 2.2 依赖安装

#### 前端依赖

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install --registry=https://registry.npmmirror.com
```

#### 后端依赖

```bash
# 进入后端目录
cd backend

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt
```

## 3. 环境配置

### 3.1 前端环境配置

前端提供了三个环境的配置文件：

- `.env.development`：开发环境配置
- `.env.testing`：测试环境配置
- `.env.production`：生产环境配置

主要配置项：

```env
# API接口地址
VITE_API_BASE_URL=http://localhost:8000/api

# 应用版本
VITE_APP_VERSION=1.0.0

# Sentry配置（可选）
VITE_SENTRY_DSN=
```

### 3.2 后端环境配置

后端提供了三个环境的配置文件：

- `.env.development`：开发环境配置
- `.env.testing`：测试环境配置
- `.env.production`：生产环境配置

主要配置项：

```env
# 环境
ENVIRONMENT=production

# 数据库配置
DB_USER=book_bridge_user
DB_PASSWORD=book_bridge_password
DB_HOST=localhost
DB_PORT=3306
DB_NAME=book_bridge

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379

# JWT密钥
SECRET_KEY=your-secret-key-for-production

# 日志级别
LOG_LEVEL=info
```

## 4. 部署方式

### 4.1 传统部署

#### 前端部署

1. **构建应用**：

```bash
# 进入前端目录
cd frontend

# 构建生产环境
npm run build:production
```

2. **部署到Nginx**：

```bash
# 复制构建产物到Nginx目录
cp -r dist/* /usr/share/nginx/html/

# 配置Nginx
cp nginx.conf /etc/nginx/conf.d/default.conf

# 重启Nginx
systemctl restart nginx
```

#### 后端部署

1. **启动应用**：

```bash
# 进入后端目录
cd backend

# 激活虚拟环境
source venv/bin/activate

# 启动应用
./deploy.sh start
```

2. **配置Supervisor**：

```bash
# 复制Supervisor配置
cp supervisor.conf /etc/supervisor/conf.d/bookbridge-backend.conf

# 重新加载Supervisor
supervisorctl reload
```

### 4.2 Docker部署

1. **构建并启动容器**：

```bash
# 在项目根目录执行
docker-compose up -d --build
```

2. **查看容器状态**：

```bash
docker-compose ps
```

3. **查看日志**：

```bash
docker-compose logs -f
```

4. **停止容器**：

```bash
docker-compose down
```

## 5. 数据库初始化

### 5.1 手动初始化

1. **创建数据库**：

```sql
CREATE DATABASE IF NOT EXISTS book_bridge CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. **执行初始化脚本**：

```bash
mysql -u root -p book_bridge < backend/init.sql
```

### 5.2 Docker自动初始化

使用Docker部署时，数据库会自动初始化，无需手动操作。

## 7. 部署常见问题

### 7.1 前端构建失败

- **问题**：依赖安装失败
- **解决方案**：检查网络连接，使用国内镜像源

### 7.2 后端启动失败

- **问题**：数据库连接失败
- **解决方案**：检查数据库配置，确保数据库服务运行正常

### 7.3 Docker容器启动失败

- **问题**：端口冲突
- **解决方案**：检查端口是否被占用，修改docker-compose.yml中的端口映射

## 8. 部署验证

1. **前端验证**：访问 http://localhost:80，确保页面正常加载
2. **后端验证**：访问 http://localhost:8000，确保返回服务运行中
3. **API验证**：访问 http://localhost:8000/api/books，确保返回图书列表

## 9. 版本管理

- **前端版本**：在package.json中定义
- **后端版本**：在main.py中定义
- **部署版本**：在.env文件中定义

## 10. 部署流程

1. **环境准备**：安装依赖，配置环境变量
2. **代码部署**：上传代码，构建应用
3. **服务启动**：启动前端和后端服务
4. **验证测试**：验证服务运行正常
5. **监控运维**：配置监控，定期检查
