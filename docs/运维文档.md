# 书驿云桥运维文档

## 1. 系统架构

### 1.1 前端架构
- **技术栈**：Vue 3 + Vue Router + Vuex + Axios + Element Plus
- **部署方式**：静态文件部署，可使用 Nginx 或 Apache 作为反向代理
- **监控体系**：
  - 日志框架：Winston + Logtail
  - 性能监控：Web Vitals + Sentry
  - 错误监控：Vue 全局错误处理 + Sentry

### 1.2 后端架构
- **技术栈**：FastAPI + SQLAlchemy + MySQL + Redis
- **部署方式**：Docker 容器化部署，可使用 Kubernetes 进行编排
- **监控体系**：
  - 日志框架：Loguru + Python JSON Logger
  - 性能监控：OpenTelemetry + Jaeger
  - 错误监控：自定义异常处理 + 日志记录

## 2. 监控体系

### 2.1 前端监控
- **日志分级**：DEBUG、INFO、WARN、ERROR、FATAL
- **日志存储**：
  - 浏览器控制台
  - Logtail 远程日志服务
- **性能指标**：
  - CLS (Cumulative Layout Shift)
  - FID (First Input Delay)
  - FCP (First Contentful Paint)
  - LCP (Largest Contentful Paint)
  - TTFB (Time to First Byte)
- **错误监控**：
  - Vue 全局错误处理
  - 未处理 Promise 拒绝捕获
  - 全局错误事件监听
  - Sentry 异常监控

### 2.2 后端监控
- **日志分级**：DEBUG、INFO、WARNING、ERROR、CRITICAL
- **日志存储**：
  - 本地文件（按级别分文件存储）
  - 支持集成 ELK 等日志管理系统
- **性能监控**：
  - API 响应时间
  - 数据库查询性能
  - 系统资源使用情况（CPU、内存、磁盘）
  - OpenTelemetry 链路追踪
- **错误监控**：
  - 自定义异常处理
  - 数据库连接失败监控
  - API 错误率监控

### 2.3 告警系统
- **告警类型**：
  - API 错误率过高
  - 数据库连接失败
  - 服务器负载过高（CPU、内存、磁盘）
  - API 响应时间过长
- **告警方式**：
  - 邮件告警
  - Slack 告警
- **告警冷却**：5分钟冷却时间，避免告警风暴

## 3. 日志管理

### 3.1 前端日志
- **日志格式**：JSON 格式，包含时间戳、级别、消息、用户ID、会话ID等信息
- **日志存储**：
  - 浏览器控制台
  - Logtail 远程日志服务

### 3.2 后端日志

- **日志存储**：
  - 本地文件（logs/info.log, logs/error.log, logs/debug.log）
  - 支持集成 ELK 等日志管理系统
- **日志轮转**：按大小轮转（100MB），保留7天（info）、30天（error）、3天（debug）
- **日志检索**：使用 ELK 等日志管理系统提供的检索功能

## 5. 部署与维护

### 5.1 前端部署
- **构建**：`npm run build`
- **部署**：将 dist 目录部署到静态文件服务器
- **配置**：Nginx 配置示例
  ```nginx
  server {
    listen 80;
    server_name your-domain.com;
    
    location / {
      root /path/to/dist;
      index index.html;
      try_files $uri $uri/ /index.html;
    }
    
    location /api {
      proxy_pass http://localhost:8000;
      proxy_set_header Host $host;
      proxy_set_header X-Real-IP $remote_addr;
      proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
  }
  ```

### 5.2 后端部署
- **依赖安装**：`pip install -r requirements.txt`
- **启动**：`uvicorn main:app --host 0.0.0.0 --port 8000`
- **Docker 部署**：
  ```dockerfile
  FROM python:3.9
  WORKDIR /app
  COPY . .
  RUN pip install -r requirements.txt
  EXPOSE 8000
  CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
  ```

### 5.3 环境配置
- **前端环境变量**：
  - `VITE_API_BASE_URL`：API 基础 URL
  - `VITE_SENTRY_DSN`：Sentry DSN
- **后端环境变量**：
  - `DATABASE_URL`：数据库连接 URL
  - `REDIS_URL`：Redis 连接 URL
  - `SECRET_KEY`：JWT 密钥
  - `SENTRY_DSN`：Sentry DSN

## 6. 监控与告警

### 6.1 监控面板
- **前端**：使用 Sentry 提供的监控面板
- **后端**：使用 Jaeger 提供的链路追踪面板和自定义监控面板

### 6.2 告警配置
- **API 错误率**：当错误率超过 10% 时触发告警
- **数据库连接**：当数据库连接失败时触发告警
- **服务器负载**：当 CPU 使用率超过 90%、内存使用率超过 90%、磁盘使用率超过 90% 时触发告警
- **响应时间**：当 API 响应时间超过 1 秒时触发告警

### 6.3 告警方式
- **邮件告警**：发送邮件到运维人员邮箱
- **Slack 告警**：发送消息到指定 Slack 频道

## 7. 安全管理

### 7.1 前端安全
- **XSS 防护**：使用 Vue 的内置 XSS 防护
- **CSRF 防护**：使用 CSRF token
- **敏感信息保护**：不存储敏感信息在前端

### 7.2 后端安全
- **SQL 注入防护**：使用 SQLAlchemy ORM
- **认证授权**：使用 JWT 认证
- **密码加密**：使用 bcrypt 加密
- **请求限流**：使用 rate limit 中间件
- **防重放攻击**：使用 nonce 和 timestamp

## 8. 备份与恢复

### 8.1 数据库备份
- **定期备份**：每天凌晨 2 点执行数据库备份
- **备份策略**：全量备份 + 增量备份
- **备份存储**：存储到云存储服务

### 8.2 日志备份
- **定期备份**：每天凌晨 3 点执行日志备份
- **备份策略**：压缩存储
- **备份存储**：存储到云存储服务

### 8.3 恢复策略
- **数据库恢复**：使用最近的备份进行恢复
- **服务恢复**：重启服务，确保配置正确

## 9. 故障处理

### 9.1 常见故障
- **API 响应缓慢**：检查数据库性能、服务器负载
- **数据库连接失败**：检查数据库服务状态、网络连接
- **服务器负载过高**：检查资源使用情况、优化代码
- **前端页面加载缓慢**：检查资源大小、网络连接

### 9.2 故障处理流程
1. **发现故障**：通过监控告警或用户反馈
2. **定位故障**：查看日志、监控面板
3. **分析故障**：确定故障原因
4. **解决故障**：执行相应的解决方案
5. **验证故障**：确认故障已解决
6. **记录故障**：记录故障原因和解决方案
